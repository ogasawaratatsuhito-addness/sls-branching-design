[1] 学習系の相談 → AIチャットへ（回答方式②）

分岐先がAIチャットなので、ここは固定回答を持たない。AIチャット（RAG/LLM）が中身を答える。下の質問分類は「AIチャットが何を答えられる必要があるか」の要件として使う。

中項目 来る質問(例) 回答方式 メモ
学習の進め方 何から始める/順番/学習ペース/どのコースから ②AIチャット 方向性そのものの相談は本来は面談C。AIチャットで一次対応し、必要なら面談導線へ（→確認点）
講義内容の不明点 Claude Code/API/Dify/MCPのやり方・エラー、講義動画の内容 ②AIチャット 旧B(講師質問)がここに統合。ログ保存の扱いをどうするか確認点
案件・お客様対応 案件の進め方/単価/クライアント対応/納品 ②AIチャット 旧Cの一部。個別性が高いものは面談へ（→確認点）

ゲート文（案）: 「これは“今取り組んでいるアクション”についての相談ですか？」→ はい＝AIチャットへ / いいえ＝[4]その他 or 該当大項目へ誘導。

[2] 添削依頼 → 添削フロー（回答方式③）

中項目 来る質問(例) 回答方式 回答/案内文(案)
作ったものを見てほしい ポートフォリオ/成果物を見てほしい ③添削フロー 「作成物を添削します。画像 または URL を添付して送ってください」
添削してほしい 提案文/プロフィール/記事の添削 ③添削フロー 同上＋「どこを見てほしいか（目的）も一言添えてください」
レビューしてほしい コード/制作物のレビュー ③添削フロー 同上

確認点: 添削の中身は誰が返す？（AIチャットで一次添削→講師確認 or 講師のみ）。画像/URLの受け取り方（Lステップで添付を受けてAIへ渡せるか）。

[3] サポート系の相談 → 情報収集→公式サポLINE（ただし“場所系”は固定回答できる）

ここが回答を決める本丸。4中項目で性質が分かれる。

3-1 エラー・不具合（回答方式①一次切り分け→ダメなら④）

まず定型のトラブルシュートを返し、解決しなければ情報収集して人へ。
- 該当する既存回答: ログイン不可/白画面/419=qa A-03(✅確定) / 動画再生・音声=A-15(✅) / なんでもサポートくん=A-10(✅) / コース解放・ロック=A-02(リンク〔要確認〕)
- 技術系のエラー（Claude Code等）は[1]講義内容（AIチャット）へ寄せるか要確認。
- 回答(案): 各テーマの確定回答（上記）をそのまま返す → 「解決しない場合は、画面のスクショと“いつ・どの画面で・何が起きたか”を添えて公式サポートLINEへ」。

3-2 予約の確認・変更（回答方式④＋一部①）

  • 入室方法の一般案内=A-11(⚠️)。予約・変更・キャンセルは予約システム/人。
  • 回答(案): 「ご予約の確認・変更・キャンセルですね。①お名前 ②予約日時 ③ご希望（確認/変更/キャンセル）を教えてください。内容をまとめて公式サポートLINEへお繋ぎします」〔要確認: 予約リンクを案内に含めるか〕。

3-3 支払い・契約（回答方式④・自動回答しない＝必ず人）

  • 旧D。解約/返金/支払いトラブル/契約乖離=qa routing D。受講期間・契約更新=A-04(❌要確認)、毎月少額引落=A-22(❌要確認)。
  • 回答(案): 「お支払い・ご契約に関するお問い合わせですね。正確にご案内するため担当者にお繋ぎします。①お名前 ②ご質問内容 を教えてください」→ 公式サポLINE/人。AIで金額・条件・可否を即答しない。

3-4 資料・アーカイブ・特典の場所（回答方式①固定回答＝即答できる本命）

ここは自己解決FAQで即答できる。来た人を待たせず解決＝価値が一番高い。
- アーカイブの場所=A-01(✅faq Q9) / アーカイブいつ=A-19(✅) / ワークシート提出=A-09(✅faq Q7) / 教材・コースの場所=A-08(⚠️) / 特典の受け取り（〔要確認〕・別導線）。
- 回答(案): 該当テーマの確定回答を即答。特典など未整備のものは「特典の受け取りは公式サポートLINEへ」。

[4] その他の相談 → 公式サポLINE（回答方式④）

  • 回答(案): 「承知しました。困っている内容をできるだけ具体的に教えてください（①状況 ②困っていること）。内容を確認して公式サポートLINEからご案内します」。

[5] AIチャット後の解決確認（回答方式⑤定型）

  • 解決した → 「解決してよかったです！また何かあればいつでもどうぞ😊」（終了）
  • まだ → 会話を要約 → 「担当者にお繋ぎします。ここまでのやりとりをまとめて共有しますね」→ 公式サポLINE。

確認点（かいてぃー／運営とすり合わせ）

  1. 面談(1on1)導線の位置: 旧Cの「方向性相談・ゴール設定1on1」は新ツリーに明示が無い。AIチャット内から面談予約へ繋ぐのか、[3]予約に入れるのか。
  2. AIチャットの中身: [1]に流す学習系の回答エンジンは、今回作るRAG（qa_knowledge_base.md＋過去ログ）か、既存の「なんでもサポートくん/講師質問」か。
  3. 講義内容(旧B)のログ保存: 旧運用は「講師質問はログが残るので便利」と誘導。AIチャットに寄せるとログ運用が変わる。どうするか。
  4. 添削の返し手: AI一次添削か講師か。画像/URL添付をLステップ→AIへ渡せるか。
  5. サポート系の即答 vs 情報収集: [3-4]場所系は即答FAQにしてよいか（分岐先は一律「情報収集→サポLINE」だが、場所系は即答が受講生体験◎）。
