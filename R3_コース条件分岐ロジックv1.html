コース条件分岐ロジック v1

  • 作成: 2026-05-30
  • 対象: ゴール設定モード（質問項目-清書v2）の回答 → 次に歩む1コースを決める
  • 出典: 2026-05-30 かいてぃーさん打ち合わせ＋たつ確定
  • 前版: かいてぃーさん共有-コース条件分岐-v0.1 / コース分岐フロー-v0.6

大前提

  • 提示するのは「次に歩む1コース」1つだけ。
  • 「ビジネス入門コース」はクライアントワーク向けの土台コース。
  • 自社（経営者・個人事業主）はビジネス入門に回さず、分野の自社向けコースへ直行する。

分岐に使う質問

  • 問2 職業（属性）… 自社 / クライアントワーク の判定
  • 問5 分野（9択）… どの系統のコースか
  • 問6 ビジネス経験 … CWで入門を挟むかの判定（自社では見ない）
  • 問7 生成AI経験 … 生成AI分野の入門/本コース出し分け
  • 問8 PC使用状況 … PC前提ゲート
  • 問9 学習時間 … コース選択には使わない（学習レポート用）

属性の定義

  • 自社: 経営者・役員 / 個人事業主
  • クライアントワーク（CW）: 会社員 / 公務員 / 学生 / パート・バイト / 主婦・主夫 / 休職中 / 求職中 / 退職・無職

分岐フロー（上から順に評価）

ゲート1: PC前提（最優先）

  • PCを持っていない、または 持っているが使い方がよく分かっていない
    → ビジネス入門コース
  • それ以外 → ゲート2へ

ゲート2: 属性

  • 自社（経営者・個人事業主）→ ゲート3A（経験年数は見ない）
  • クライアントワーク → ゲート3B
